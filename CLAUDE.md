# 프로젝트 규칙

## ⛔ 빌드 트리거 — 절대 규칙
GitHub Actions 빌드(태그 푸쉬 → EXE 배포)는 **사용자가 명시적으로 요청할 때만** 실행한다.
버그가 있다는 것을 알고 있는 상태에서 빌드 트리거를 하지 않는다.
태그를 붙여야 할 경우 로컬에 태그만 생성하고, 사용자 요청 없이 `git push --tags`는 절대 하지 않는다.

**위반 사례:** 맥 테스트 전에 v2.13.9 푸시 — 이런 행동 절대 반복하지 말 것.

## ⛔ 브로제이(Broj) CRM — API 없음
브로제이(crm.broj.co.kr)는 **공개 API를 제공하지 않는다.**
락카 데이터는 반드시 Selenium으로 DOM을 직접 파싱해야 한다.
"API 호출", "REST API", "공개 API" 등을 대안으로 절대 제안하지 말 것.
