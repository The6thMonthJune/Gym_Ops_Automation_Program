# Gym Ops Automation Program

헬스장 현장의 반복 업무를 자동화하는 데스크톱 프로그램입니다.  
매일 수동으로 처리하던 엑셀 파일 생성, 매출/지출 보고, 결제 등록을 버튼 클릭 하나로 처리합니다.

---

## 배경 및 문제 정의

헬스장에서는 매일 다음과 같은 작업이 반복됩니다.

| 업무 | 기존 방식 | 문제점 |
|------|-----------|--------|
| 데일리 엑셀 파일 생성 | 전날 파일을 수동 복사 후 파일명 변경 | 날짜 오기입, 파일 누락 위험 |
| 매출 보고 문구 작성 | 엑셀 열어서 셀 값 확인 후 직접 타이핑 | 매일 같은 포맷의 단순 반복 작업 |
| 결제 내역 이중 등록 | 데일리·총매출 파일 각각 수동 입력 | 누락·중복 위험, 파일이 열린 상태에서는 저장 불가 |
| 지출 내역 등록 | 데일리·총매출 파일 각각 수동 입력 + 카톡 수동 작성 | 누락 위험, 매번 양식 확인 필요 |
| 카톡 보고 | 매번 정해진 양식으로 직접 작성 | 오타, 누락 위험 |

이 프로젝트는 위 흐름을 분석하고, **가장 빈도가 높고 자동화 가치가 큰 작업부터 MVP로 구현**하는 방식으로 시작했습니다.

## 주요 기능

### 1. 다음 날짜 데일리 파일 자동 생성
- 오늘 파일 등록 시점에 내일 날짜 파일을 즉시 선제 생성
- 파일명에서 날짜를 자동 인식해 하루 뒤 파일명으로 복사본 생성
- 월말/연말 날짜 경계 자동 처리 (`3.31` → `4.1`, `12.31` → `1.1`)
- 자정이 지나면 타이머로 날짜 변경을 감지해 오늘 파일 자동 탐색·등록

### 2. 매출 보고 문구 자동 생성
- 데일리 엑셀 파일의 `데일리매출` 시트에서 셀 값 자동 읽기
- 파일명에서 날짜를 추출해 보고 헤더에 자동 반영
- 생성된 문구를 클립보드에 복사

```
4.11 금

현금:
0원

카드:
336,000원

계좌:
0원

총합:
336,000원
```

### 3. 결제 입력 자동화 (v1.1: 센터/레슨 구분 추가)
결제 1건 발생 시 아래 작업을 한 화면에서 처리합니다.

| 작업 | 처리 방식 |
|------|----------|
| 데일리 엑셀 입력 | 자동 (xlwings — 파일 열림 여부 무관) |
| 총매출 엑셀 입력 | 자동 (xlwings — 암호 해제 포함, 올바른 월 시트 자동 탐색) |
| 카톡 매출보고 문구 생성 | 자동 생성 후 복사 버튼 제공 |
| 중복 방지 | 같은 날짜·회원명·금액 존재 시 경고 |

**센터 / 레슨 구분 (v1.1):**  
같은 엑셀 시트 내에서 센터 매출은 B열, 레슨 매출은 P열부터 기록합니다.  
종목 선택 시 PT는 자동으로 레슨으로 pre-select되며, 사용자가 직접 변경도 가능합니다.

```
📌매출보고📌

신규/재등 : 신규
성함 : 홍길동
회원권 : PT 20회
금액 : 1,200,000원
결제방법: 계좌이체
특이사항:
```

### 4. 지출 입력 자동화 (v1.2)
지출 1건 발생 시 아래 작업을 한 화면에서 처리합니다.

| 작업 | 처리 방식 |
|------|----------|
| 데일리 지출 시트 입력 | 자동 (번호 자동 증가) |
| 총매출 월별 지출 시트 입력 | 자동 (해당 월 시트 자동 탐색) |
| 카톡 지출보고 문구 생성 | 자동 생성 후 복사 버튼 제공 |

지출 구분: 지점 비품 / 수수료 / 주차비 / 복리후생 / 운반비 / 당직비 / 홍보 / 회식대 / 간식대 / 구인 / 세금 / 환불 / 기타

```
📌지출보고📌

일자: 2026.03.25
구분: 지점 비품
지출내용: 청테이프 2개
금액 : ₩4,960
결제방법: 카드
담당자: 인포
거래처: 삼육오마트 일산 중산점
```

### 5. 카톡 자동 전송 (v2.2)
결제/지출 보고 문구 생성 후 버튼 하나로 카카오톡 방에 자동 전송합니다.

| 항목 | 내용 |
|------|------|
| 전송 대상 | 직원방 / 알바방 선택 |
| 동작 방식 | 센터폰(Android)에서 메신저봇R 앱이 HTTP 서버(포트 9094)를 열고 대기 |
| PC → 폰 통신 | Python `urllib` → HTTP POST `{ target, msg }` |
| 폰 → 카톡 | 메신저봇R `Api.replyRoom()` 으로 대상 방에 전송 |
| 기술 포인트 | Rhino JS 엔진에서 `java.net.ServerSocket` 직접 구동, Java String / JS String 타입 차이 해결 |

### 6. 메인 화면 대시보드 (v2.4)
앱 실행 시 오늘 매출을 즉시 파악하고, 모든 기능에 한 번에 접근할 수 있는 대시보드 형태로 개편했습니다.

| 구성 요소 | 내용 |
|----------|------|
| 오늘 매출 요약 카드 | 데일리 파일에서 현금·카드·계좌·총합을 자동으로 읽어 표시 |
| 새로고침 버튼 | 파일 변경 후 수동으로 값 갱신 |
| 보고 문구 복사 | 카드 내 버튼 한 번으로 클립보드에 복사 |
| 회원 현황 보고 | 브로제이 엑셀 기반 전체 회원 상태 집계 후 클립보드 복사 (활성/만료/예정/임박/홀딩/미등록) |
| 바로가기 버튼 4개 | 매출 입력 / 지출 입력 / 내역 조회 / 락카 현황 |

### 7. 신규 회원 유입경로 추적 (v2.5)
신규 회원 등록 시 유입경로를 함께 기록하고, 월별 통계를 자동 집계합니다.

| 항목 | 내용 |
|------|------|
| 유입경로 분류 | 현수막·지인 소개·온라인·기타 등 선택 입력 |
| 월별 집계 | 경로별 신규 회원 수를 월 단위로 자동 집계 |
| 보고 문구 생성 | 대표 보고용 월별 유입경로 현황 문구 자동 생성 후 클립보드 복사 |

### 8. 회원 현황 시각화 + 변화 추세 그래프 (v2.7)
회원 현황 보고 버튼 클릭 시 팝업으로 시각화 화면이 열립니다.

| 항목 | 내용 |
|------|------|
| 상태 카드 | 활성·만료·홀딩·예정·미등록·총 인원을 색상 카드로 표시 |
| 추세 그래프 | 전일·전주·전월 중 선택해 회원 수 변화를 꺾은선 그래프로 표시 |
| 데이터 축적 | DB 업데이트 시 SQLite에 일별 스냅샷 자동 저장 |
| 복사 기능 | 이미지 클립보드 복사 / 보고 문구 텍스트 복사 |

### 9. 네이트온 팀룸 매출 보고 자동 전송 (v2.8)
앱이 실행 중인 상태에서 지정 시각에 일일 매출 보고 문구를 네이트온 팀룸으로 자동 전송합니다.

| 항목 | 내용 |
|------|------|
| 평일 전송 시각 | 23:50 (마감 직전) |
| 주말·공휴일 전송 시각 | 07:30 (08:00 마감 기준) |
| 공휴일 감지 | `holidays` 라이브러리 — 대체공휴일 포함 자동 감지 |
| 전송 방식 | 네이트온 인커밍 웹훅 HTTP POST (`{"text": "..."}`) |
| 중복 방지 | 하루 1회만 전송, 재전송 없음 |
| 설정 | 설정 화면에서 웹훅 URL 입력 후 저장 |

### 10. 헬스권 만료 임박 회원 조회 (v2.9)
문자 마케팅을 위해 회원권 만료가 임박한 회원의 이름과 전화번호를 빠르게 추출합니다.

| 항목 | 내용 |
|------|------|
| 필터 기준 | 만료까지 N일 이하 (기본 70일, 1~365일 조정 가능) |
| 표시 정보 | 이름·연락처·잔여일·보유 이용권, 잔여일 오름차순 정렬 |
| 강조 표시 | 잔여일 9일 이하 회원은 빨간색으로 강조 |
| 전화번호 복사 | 버튼 한 번으로 전체 번호를 줄바꿈 구분 복사 (문자 수신 목록에 바로 붙여넣기) |

### 11. 총매출 월 시트 자동 생성 (v2.10)
매월 첫 결제 시 총매출 파일에 해당 월 시트가 없어도 자동으로 생성됩니다.

| 항목 | 내용 |
|------|------|
| 생성 방식 | 전월 시트를 복사해 새 시트 생성 (서식·수식 유지) |
| 감지 시점 | 앱 시작 시 + 자정 이후 월 전환 감지 시 자동 확인 |
| 효과 | 매월 초 수동으로 시트를 복사·생성하는 반복 작업 제거 |

### 12. 락카 만료자 명단 복사 분화 (v2.10)
락카 만료자 명단에서 목적에 맞는 형식으로 즉시 복사할 수 있습니다.

| 버튼 | 복사 내용 |
|------|----------|
| 📞 전화번호 | 만료자 전화번호만 줄바꿈 구분 (문자 수신 목록에 바로 붙여넣기) |
| 📋 명단 | 이름·락카번호·이용권 만료 여부 + 자물쇠·물품·연락·락카 회수 체크리스트 |

### 13. 신규 회원 거주지 선택 입력 (v2.10.4)
신규 회원 등록 시 거주지를 텍스트로 직접 입력하는 대신, 미리 설정한 아파트 단지 목록에서 선택합니다.

| 항목 | 내용 |
|------|------|
| 단지 관리 | 설정 화면에서 아파트 단지 추가·삭제 |
| 신규 등록 시 | 드롭다운으로 단지 선택 (직접 입력 옵션 유지) |
| 보고서 효과 | 거주지역별 현황 시트의 데이터 일관성 향상 |

### 14. 월 목표 카운트다운 (v2.11)
데일리 파일에서 센터·레슨 매출을 읽어 월간 누적 목표 달성 현황을 보고 문구로 생성합니다.

| 항목 | 내용 |
|------|------|
| 자동 집계 | 데일리 파일의 센터/레슨 섹션 합산 |
| 누적 이어받기 | JSON에 기준선 저장, 다음 날 열면 어제까지 누적 + 오늘 데일리 합산 |
| 보고 문구 | 센터/피티/총합 현황 + 목표 잔여액 표시 |
| 목표 설정 | ⚙ 설정에서 센터·피티 월 목표금액 입력 |

### 15. 외국인 회원 SMS 알림 (v2.12)
브로제이에서 제공하지 않는 외국인 회원 정보를 별도 관리하고, 공휴일 안내 및 만료 락카 문자를 자동 발송합니다.

| 기능 | 내용 |
|------|------|
| 외국인 회원 관리 | 이름·전화번호 직접 등록, DB 업데이트 시 활성/만료 상태 자동 동기화 |
| 공휴일 단축운영 안내 | 매월 1일 자동 팝업 → 외국인 활성 회원에게 한/영 이중 언어 문자 발송 |
| 임시공휴일 | 지방선거 등 `holidays` 라이브러리 미포함 공휴일은 다이얼로그에서 수동 추가 |
| 만료 락카 안내 | 전체 만료 락카 회원 대상, 외국인은 한/영 이중 언어·한국인은 한국어만 발송 |
| 신규 만료 자동 탐지 | DB 업데이트 시 이전 스냅샷 비교 → 새로 만료된 회원 발생 시 즉시 알림 |
| 테스트 발송 | 설정에 등록한 번호로 실제 발송 전 문구 확인 |
| 발송 채널 | 센터폰의 SMS Gateway 앱(android-sms-gateway) HTTP REST API |

### 16. 파일 경로 자동 저장
처음 한 번만 파일을 선택하면, 이후 앱을 재시작해도 경로가 자동으로 복원됩니다.  
경로·비밀번호·센터폰 IP 설정은 `%APPDATA%\리와인드자동화\settings.json`에 저장되며 Git에 올라가지 않습니다.

## 실행 방법

```bash
# 가상환경 생성 및 의존성 설치
python -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.txt

# 앱 실행
.venv\Scripts\python.exe run.py

# 테스트
.venv\Scripts\python.exe -m pytest tests/ -v
```

> PowerShell에서 activate가 막히는 경우:  
> `Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass` 후 재시도하거나,  
> 위처럼 `python.exe`를 직접 호출하세요.

### 초기 설정 (헬스장 PC 첫 실행 시)

1. 프로그램 실행
2. 데일리 파일, 총매출 파일 각각 선택 (이후 자동 복원)
3. **⚙ 설정** → 총매출 파일 비밀번호 + 지출 시트 이름 입력 (`데일리지출`)
4. 이후 재시작해도 경로·설정 자동 복원, 오늘 날짜 파일 자동 생성
