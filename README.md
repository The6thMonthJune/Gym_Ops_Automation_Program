# Gym Ops Automation Program

헬스장 현장의 반복 업무를 자동화하는 데스크톱 프로그램입니다.  
매일 수동으로 처리하던 엑셀 파일 생성, 매출 보고 문구 작성, 결제 등록을 버튼 클릭 하나로 처리합니다.

---

## 배경 및 문제 정의

헬스장에서는 매일 다음과 같은 작업이 반복됩니다.

| 업무 | 기존 방식 | 문제점 |
|------|-----------|--------|
| 데일리 엑셀 파일 생성 | 전날 파일을 수동 복사 후 파일명 변경 | 날짜 오기입, 파일 누락 위험 |
| 매출 보고 문구 작성 | 엑셀 열어서 셀 값 확인 후 직접 타이핑 | 매일 같은 포맷의 단순 반복 작업 |
| 결제 내역 이중 등록 | 데일리·총매출 파일 각각 수동 입력 | 누락·중복 위험, 파일이 열린 상태에서는 저장 불가 |
| 카톡 매출 보고 | 매번 정해진 양식으로 직접 작성 | 오타, 누락 위험 |

이 프로젝트는 위 흐름을 분석하고, **가장 빈도가 높고 자동화 가치가 큰 작업부터 MVP로 구현**하는 방식으로 시작했습니다.

---

## 주요 기능

### 1. 다음 날짜 데일리 파일 자동 생성
- 기준 엑셀 파일을 선택하면 파일명에서 날짜를 자동 인식
- 버튼 클릭 시 하루 뒤 날짜로 파일명을 바꿔 복사본 생성
- 월말/연말 날짜 경계 자동 처리 (`3.31` → `4.1`, `12.31` → `1.1`)

### 2. 매출 보고 문구 자동 생성
- 데일리 엑셀 파일의 `데일리매출` 시트에서 셀 값 자동 읽기
- 파일명에서 날짜를 추출해 보고 헤더에 자동 반영
- 생성된 문구를 클립보드에 복사

```
4.11 금

현금:
0원

카드:
336,000원

계좌:
0원

총합:
336,000원
```

### 3. 결제 입력 및 카톡 보고 문구 한번에 처리
결제 1건 발생 시 아래 4가지 작업을 한 화면에서 처리합니다.

| 작업 | 처리 방식 |
|------|----------|
| 데일리 엑셀 입력 | 자동 (xlwings — 파일 열림 여부 무관) |
| 총매출 엑셀 입력 | 자동 (xlwings — 암호 해제 포함, 올바른 월 시트 자동 탐색) |
| 카톡 매출보고 문구 생성 | 자동 생성 후 복사 버튼 제공 |
| 브로제이 입력 | 공식 API 미확인 — 미구현 |

**중복 입력 방지:** 같은 날짜·회원명·금액이 이미 존재하면 경고 후 강제 입력 여부를 확인합니다.

카톡 보고 양식:
```
📌매출보고📌

신규/재등 : 신규
성함 : 홍길동
회원권 : 헬스 3개월
금액 : 336,000원
결제방법: 카드
특이사항:
```

### 4. 파일 경로 자동 저장
처음 한 번만 파일을 선택하면, 이후 앱을 재시작해도 경로가 자동으로 복원됩니다.  
설정은 `%APPDATA%\리와인드자동화\settings.json`에 저장됩니다.

## 프로젝트 구조

```
Gym_Ops_Automation_Program/
├── src/
│   ├── main.py                      # 앱 진입점
│   ├── ui/
│   │   ├── main_window.py           # 메인 화면 (PySide6)
│   │   └── payment_dialog.py        # 결제 입력 다이얼로그
│   ├── services/
│   │   ├── daily_file_service.py    # 다음 날짜 파일 복사 생성
│   │   ├── sales_report_service.py  # 엑셀 읽기 + 보고 문구 생성
│   │   ├── entry_service.py         # 결제 내역 엑셀 쓰기 (xlwings)
│   │   └── total_sales_service.py   # 총매출 파일 월별 시트 탐색
│   ├── core/
│   │   └── file_naming.py           # 파일명 날짜 파싱 및 변환
│   └── config/
│       ├── constants.py             # 셀 주소, 비밀번호 등 상수
│       └── settings.py              # 파일 경로 영구 저장/불러오기
├── tests/
│   ├── test_file_naming.py
│   └── test_sales_report_service.py
├── conftest.py
└── requirements.txt
```

**레이어 설계 원칙:**  
`core`(날짜 파싱) → `services`(파일/엑셀 처리) → `ui`(화면) 순으로 의존 방향을 단방향으로 유지해 각 계층을 독립적으로 테스트할 수 있습니다.

## 실행 방법

```bash
# 가상환경 생성 및 의존성 설치
python -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.txt

# 앱 실행 (-m 옵션 필수: src.* 절대 import 경로 기준)
.venv\Scripts\python.exe -m src.main

# 테스트
.venv\Scripts\python.exe -m pytest tests/ -v
```

> PowerShell에서 activate가 막히는 경우:
> `Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass` 후 재시도하거나,
> 위처럼 `python.exe`를 직접 호출하세요.

### 초기 설정 (헬스장 PC 첫 실행 시)

1. 프로그램 실행
2. **기준 엑셀 파일** — 복사 기준이 되는 데일리 양식 파일 선택
3. **데일리 엑셀 파일** — 오늘 날짜 데일리 파일 선택
4. **총매출 엑셀 파일** — 월별 총매출 파일 선택
5. `src/config/constants.py`의 `TOTAL_SALES_PASSWORD`에 총매출 파일 비밀번호 입력

이후 재시작해도 경로가 자동 복원됩니다.

## EXE 빌드 (헬스장 PC 배포)

```bash
.venv\Scripts\python.exe -m pip install pyinstaller
.venv\Scripts\python.exe -m PyInstaller --onefile --windowed --name "리와인드자동화" src/main.py
```

`dist/리와인드자동화.exe` 생성 후, 해당 파일 하나만 헬스장 PC에 복사하면 됩니다.  
Python 설치 불필요. Excel은 반드시 설치되어 있어야 합니다 (xlwings 의존).

## 확장 로드맵

```
v1 (현재)  파일 자동 생성 + 매출 보고 문구 + 결제 이중 등록 자동화
v2         네이트온 자동 전송, 로그 기록
v3         다중 지점 대응, 회원 계약서 OCR, 문의 자동화
```

MVP를 먼저 현장에 도입해 피드백을 수집하고, 실제 사용 빈도와 불편함을 기준으로 다음 기능의 우선순위를 결정하는 방식으로 개발을 이어갈 계획입니다.
