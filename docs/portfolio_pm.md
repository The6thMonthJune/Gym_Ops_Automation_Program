# 포트폴리오 — PM 관점
# Gym Ops Automation Program

> 헬스장 현장 운영 비효율을 직접 발굴하고 자동화 프로그램으로 해결한 프로젝트  
> 개인 프로젝트 | 2025.12 ~ 2026.04

---

## 한 줄 요약

헬스장 직원으로 근무하며 매일 반복되는 엑셀·카톡 업무를 직접 분석하고, 우선순위를 정해 단계적으로 자동화 프로그램을 만들어 현장에 배포했습니다.

---

## 배경 — 왜 이 문제를 선택했나

헬스장에서는 매일 같은 작업이 반복됩니다.

- 전날 엑셀 파일을 복사해서 날짜만 바꾸기
- 엑셀 숫자를 보면서 카톡에 보고 문구 타이핑
- 결제 1건이 발생하면 데일리 파일과 총매출 파일에 **각각** 수동 입력

이 작업들은 단순하지만 매일 반복되고, 실수가 나면 데이터 불일치로 이어집니다. 특히 **총매출 파일이 Excel에 열려 있으면 저장이 안 된다**는 문제가 실무에서 가장 큰 불편이었습니다.

---

## 문제 정의 및 우선순위 선정

모든 반복 업무를 나열한 뒤, **빈도 × 자동화 난이도** 기준으로 MVP 범위를 결정했습니다.

| 업무 | 일 빈도 | 자동화 난이도 | MVP 포함 |
|------|---------|---------------|----------|
| 데일리 파일 생성 | 1회 | 낮음 | ✅ |
| 매출 보고 문구 | 1회 | 낮음 | ✅ |
| 결제 등록 (데일리+총매출) | 2~10회 | 중간 | ✅ |
| 지출 등록 (데일리+총매출) | 1~5회 | 중간 | ✅ v1.2 |
| 락카 관리 | 월 1~2회 | 높음 | 🔜 v2.0 |
| 네이트온 자동 전송 | 1회 | 높음 | 🔜 v1.3 |
| 카카오톡 자동 전송 | 1회 | 매우 높음 | ❌ 비공개 API |

MVP에서는 **매일 발생하고 실수 위험이 있는 작업**을 먼저 해결했습니다.

## 기능 구현 과정과 의사결정

### v1.0 — 핵심 3가지 자동화

처음에는 가장 빈도가 높은 3가지를 구현했습니다.

1. 파일명에서 날짜를 인식해 하루 뒤 파일을 자동 생성
2. 엑셀 셀 값을 읽어 카톡 매출 보고 문구 자동 생성
3. 결제 1건을 입력하면 데일리·총매출 파일에 동시 기록

이때 가장 큰 기술적 문제는 **Excel 파일이 열려 있으면 저장이 안 된다**는 점이었습니다. openpyxl은 파일이 열려 있으면 `PermissionError`를 냅니다. Excel COM 인터페이스를 활용하는 xlwings로 전환해 해결했습니다.

### v1.0 → 세부 개선

현장에서 실제로 사용하면서 추가로 발견한 불편:

- **파일 경로를 매번 다시 선택해야 함** → AppData에 경로 자동 저장
- **비밀번호가 코드에 하드코딩** → 보안 문제 → GUI 설정 다이얼로그로 분리
- **드래그&드롭 지원 안 됨** → 파일탐색기에서 바로 끌어다 놓을 수 있도록 개선

### v1.1 — 센터/레슨 구분

현장에서 새로운 요구가 생겼습니다.

> "PT 결제는 센터 매출이 아니라 레슨 매출로 들어가야 해요. 엑셀에서 다른 열에 써야 해요."

같은 시트 안에서 센터(B열)와 레슨(P열)을 구분해서 기록해야 합니다. 자동 판별 알고리즘 대신 **사용자가 라디오 버튼으로 직접 선택**하는 방식을 택했습니다. PT 종목 선택 시 자동 pre-select로 실수는 줄이되, 최종 결정권은 사람에게 남겼습니다.

### v1.2 — 지출보고

> "지출이 생길 때마다 데일리·총매출 파일에 각각 입력하고 카톡방에도 보고해야 하는데 너무 번거로워요."

매출 입력과 동일한 구조로 지출 입력 기능을 추가했습니다. 결제 대신 지출 구분(비품/수수료/주차비 등)과 거래처 필드를 가진 폼을 만들고, 카톡 지출보고 문구도 자동 생성합니다.

## 기술 결정의 PM적 판단

### 카카오톡 자동 전송 — 구현하지 않은 이유

처음에는 카톡 자동 전송을 원했습니다. 하지만 카카오 비즈니스 API는 비즈니스 계정이 필요하고, 일반 채팅방은 공식 API가 없습니다. 비공식 방법은 정책 위반 위험이 있어 **클립보드 복사 방식**을 유지했습니다.

> "완벽한 자동화보다 안전하게 쓸 수 있는 자동화가 낫다."

### 락카 관리 — Playwright vs 엑셀 내보내기

락카 관리 자동화를 위해 브로제이 CRM 웹을 Playwright로 자동 스크래핑하는 방안을 먼저 검토했습니다. 하지만:

- 계정 정지·IP 밴 위험
- 페이지 구조가 바뀌면 코드도 깨짐
- 브로제이에 엑셀 내보내기 기능이 이미 존재

따라서 **엑셀 내보내기 파싱 방식으로 pivot**했습니다. 더 안전하고, 유지보수 비용도 낮습니다.

## 성과

- 현장 배포 후 실사용 중
- 결제 등록 소요 시간: 건당 5~10분 → 1분 이내
- 데이터 이중 입력 누락 사고 제로
- 파일이 열려 있어도 정상 작동으로 업무 흐름 단절 없음

## 이 프로젝트에서 배운 것

**문제는 현장에 있다.**  
기획서가 아니라 내가 직접 겪은 불편에서 시작했기 때문에, 무엇이 진짜 문제인지 명확히 알고 있었습니다. 덕분에 기능 하나하나의 우선순위를 빠르게 결정하고 실용적인 수준에서 구현할 수 있었습니다.

**완벽보다 동작하는 것이 먼저.**  
카카오톡 자동 전송, Playwright 스크래핑 같은 기술적으로 흥미로운 방법 대신, 실제로 안전하게 동작하는 방법을 선택했습니다. 현장에서 쓰이는 프로그램은 화려함보다 신뢰성이 중요합니다.
